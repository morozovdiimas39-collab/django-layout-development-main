# Telegram-бот с Gemini 2.5 Flash

Отдельный бот только для автоматической переписки с клиентами: каждый текст пользователя обрабатывает **Gemini 2.5 Flash** и ответ уходит в чат.

- Один бот = только диалог (без кнопок и заявок).
- Системная инструкция задаёт роль: помощник школы актёрского мастерства и ораторского искусства.
- Команда `/start` — приветствие; любое текстовое сообщение — ответ от Gemini.

## Переменные окружения в Yandex Cloud

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `TELEGRAM_BOT_TOKEN_GEMINI` или `TELEGRAM_BOT_TOKEN` | да | Токен **другого** Telegram-бота (не тот, что для заявок). |
| `GEMINI_API_KEY` | да | Ключ Google AI (Gemini API). |
| `TELEGRAM_GEMINI_ADMIN_CHAT_ID` или `TELEGRAM_ADMIN_CHAT_ID` | нет | **ID чата**, куда дублируются все сообщения (что написал пользователь и что ответил бот). Так вы видите всю переписку. Узнать свой chat_id: напишите боту @userinfobot в Telegram. |
| `METRIKA_CONVERSION_URL` | нет | URL облачной функции **metrika-conversion** (например `https://functions.yandexcloud.net/d4e1l3lvret5b8ora95c`). Если задан — при фразах вроде «хочу», «записаться», «перезвони» в Метрику уходит офлайн-конверсия с `client_id=telegram_<chat_id>`. В счётчике должна быть цель с идентификатором из `METRIKA_GOAL_TELEGRAM`. |
| `METRIKA_GOAL_TELEGRAM` | нет | Идентификатор цели в Яндекс.Метрике для конверсий из этого бота (по умолчанию `TELEGRAM_LEAD`). |
| `TELEGRAM_PROXY_URL` | нет | Прокси для запросов к Telegram API (например `http://proxy:port`). |
| `GEMINI_PROXY_URL` | нет | Прокси для запросов к Gemini API. Если не задан — используется `TELEGRAM_PROXY_URL`. |
| `TELEGRAM_START_PHOTO_URL` или `TELEGRAM_START_PHOTO_FILE_ID` | нет | При **/start** бот отправит это **фото** с приветственным текстом в подписи. Укажите публичный URL картинки или file_id (если фото уже отправляли этому боту). |
| `TELEGRAM_START_VIDEO_URL` или `TELEGRAM_START_VIDEO_FILE_ID` | нет | То же для **видео** на /start (если задано и фото не задано — отправится видео с текстом в подписи). |

## Фото и видео с текстом на /start

Если задана переменная **`TELEGRAM_START_PHOTO_URL`** (или **`TELEGRAM_START_PHOTO_FILE_ID`**), то по команде /start бот отправит это фото, а приветственный текст будет в подписи (caption). Если задано только **`TELEGRAM_START_VIDEO_URL`** (или file_id видео) — отправится видео с тем же текстом в подписи. Иначе по /start уходит только текст, как раньше.

## Деплой в Yandex Cloud Functions

1. Создайте новую функцию (или склонируйте существующую).
2. Рантайм: **Python 3.11** (или 3.10).
3. Точка входа: `index.handler`.
4. Загрузите код: папка `telegram-bot-gemini` (файлы `index.py`, `requirements.txt`).
5. В настройках функции укажите переменные окружения выше.
6. После деплоя скопируйте URL функции (например `https://functions.yandexcloud.net/...`).

## Webhook Telegram

После деплоя повесьте webhook **этого** бота на URL вашей функции:

```bash
curl -X POST "https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN_GEMINI>/setWebhook?url=<URL_ВАШЕЙ_ФУНКЦИИ>"
```

Пример:

```bash
curl -X POST "https://api.telegram.org/bot123456:ABC/setWebhook?url=https://functions.yandexcloud.net/d4eXXXXXXXX"
```

Проверка webhook:

```bash
curl "https://api.telegram.org/bot<TOKEN>/getWebhookInfo"
```

## Видеть все сообщения

Задайте в функции переменную **`TELEGRAM_GEMINI_ADMIN_CHAT_ID`** (или **`TELEGRAM_ADMIN_CHAT_ID`**) — числовой ID чата, куда бот будет присылать копию каждой переписки: что написал пользователь и что ответил бот. Узнать свой chat_id: напишите в Telegram боту [@userinfobot](https://t.me/userinfobot) — он вернёт ваш ID. Этот же ID укажите в настройках функции.

## Передача конверсий в Метрику

Если задан **`METRIKA_CONVERSION_URL`** (URL вашей облачной функции metrika-conversion), при сообщениях с намерением записаться («хочу», «записаться», «перезвони» и т.п.) бот отправляет в Метрику офлайн-конверсию с `client_id=telegram_<chat_id>`. В счётчике Метрики должна быть создана цель с идентификатором из **`METRIKA_GOAL_TELEGRAM`** (по умолчанию `TELEGRAM_LEAD`). У функции metrika-conversion должна быть настроена переменная `YANDEX_METRIKA_TOKEN`.

## Периодические сообщения от бота (по расписанию)

Чтобы бот **сам периодически что-то писал** (например, раз в день в группу или канал), используйте файл **`cron_send.py`** в этой же папке.

1. В Yandex Cloud создайте **вторую функцию** (или отдельный сервис) с точкой входа **`cron_send.handler`** и тем же кодом/артефактом (папка telegram-bot-gemini).
2. Создайте **триггер по расписанию** (Timer/Cron), который вызывает эту функцию с нужной частотой (например, раз в день: `cron: 0 10 * * *` — каждый день в 10:00).
3. В настройках функции задайте переменные:
   - **`TELEGRAM_BOT_TOKEN_GEMINI`** (или `TELEGRAM_BOT_TOKEN`) — тот же токен бота.
   - **`TELEGRAM_CRON_CHAT_ID`** — ID чата или канала, куда слать сообщение (число или `@channelname` для публичного канала; бот должен быть в чате/канале).
   - **`TELEGRAM_CRON_MESSAGE`** — текст сообщения (по умолчанию: «Напоминание от бота 👋»).

После этого по расписанию бот будет отправлять это сообщение в указанный чат. Чтобы слать разным людям, нужна база chat_id (сохранять при /start) и в cron_send читать её и слать в цикле — это отдельная доработка.

## Отличие от telegram-bot-layout

- **telegram-bot-layout** — бот для заявок: кнопки (дозвонились, записался, нецелевой и т.д.) и обновление статусов в БД; при желании туда же добавлен ответ на текст через Gemini.
- **telegram-bot-gemini** — отдельный бот только для чата с клиентами через Gemini 2.5 Flash, без кнопок и без обращения к leads API.

Используйте два разных бота и два разных токена, если нужны и заявки, и автоответы в одном проекте.
