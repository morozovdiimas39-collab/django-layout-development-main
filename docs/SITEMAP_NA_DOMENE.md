# Sitemap на домене (казбек-меретуков.рф/sitemap.xml)

Чтобы **казбек-меретуков.рф/sitemap.xml** отдавал XML с того же домена (не редирект, не URL функции), трафик домена должен идти через **API Gateway** в Yandex Cloud.

## Два шага в Yandex Cloud

### 1. API Gateway со спецификацией

- Открой в корне репо файл **`api-gateway-main-domain.yaml`**.
- Замени **`BUCKET_NAME`** на имя бакета, откуда сейчас раздаётся сайт.
- Замени **`SERVICE_ACCOUNT_ID`** на ID сервисного аккаунта с доступом к этому бакету (если при сохранении будет 400 — возьми в разделе «Сервисные аккаунты» в Yandex Cloud).
- В консоли Yandex Cloud: **API Gateway** → создать API или открыть существующий → вставить содержимое этого файла → сохранить.

### 2. Привязать домен к шлюзу

- В настройках домена **казбек-меретуков.рф** (там, где он привязан к бакету/хостингу) укажи, что трафик идёт в **этот API Gateway**, а не напрямую в бакет.

После этого:

- **казбек-меретуков.рф** — сайт из бакета (как сейчас).
- **казбек-меретуков.рф/sitemap.xml** — XML с домена, из функции sitemap.

Если раньше при привязке домена к шлюзу страница была чёрной — скорее всего, в шлюзе была только маршрутизация sitemap, без отдачи сайта из бакета. Нужна именно полная спецификация из **api-gateway-main-domain.yaml** (и сайт из бакета, и sitemap из функции).
