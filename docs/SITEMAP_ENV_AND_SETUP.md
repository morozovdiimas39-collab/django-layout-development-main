# Sitemap: переменные окружения и что нужно настроить

## 1. Функция sitemap (Yandex Cloud Functions)

Переменные окружения задаются в консоли Yandex Cloud: Serverless → Функции → sitemap → Редактировать → Переменные окружения.

| Переменная | Обязательная | Что указать |
|------------|--------------|-------------|
| **DATABASE_URL** | Да | Строка подключения к PostgreSQL, например: `postgresql://USER:PASSWORD@HOST:6432/DATABASE?sslmode=require` (те же данные, что у функции gallery или у приложения с блогом). |
| **DB_SEARCH_PATH** | Нет | Схемы БД через запятую. По умолчанию в коде: `t_p90119217_django_layout_develo, public`. Менять только если у тебя другие схемы. |

Без корректного **DATABASE_URL** функция не сможет взять список статей блога — в sitemap будут только статические страницы (или ошибка).

---

## 2. API Gateway (чтобы sitemap открывался по домену)

Переменных окружения у шлюза нет. Нужно вручную подставить значения в файл **api-gateway-main-domain.yaml** перед загрузкой в API Gateway:

| В файле написано | Заменить на |
|------------------|-------------|
| **BUCKET_NAME** | Имя бакета в Object Storage, откуда раздаётся сайт (тот же бакет, куда деплоится фронт). |
| **SERVICE_ACCOUNT_ID** | ID сервисного аккаунта в Yandex Cloud, у которого есть доступ к этому бакету (например, роль `storage.editor` на бакет). Узнать: IAM → Сервисные аккаунты → нужный аккаунт → скопировать ID. |

Дальше: в консоли создать/обновить API Gateway, вставить в него содержимое этого yaml (уже с подставленными BUCKET_NAME и SERVICE_ACCOUNT_ID), сохранить. Потом привязать домен казбек-меретуков.рф к этому шлюзу.

---

## 3. Кратко

- **Функция sitemap:** в настройках функции обязательно задай **DATABASE_URL** (и при необходимости **DB_SEARCH_PATH**).
- **Домен и sitemap на домене:** в **api-gateway-main-domain.yaml** подставь **BUCKET_NAME** и **SERVICE_ACCOUNT_ID**, залей спецификацию в API Gateway и привяжи к нему домен.
