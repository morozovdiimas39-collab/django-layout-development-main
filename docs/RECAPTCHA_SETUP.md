# Невидимая капча reCAPTCHA v3 на формах

На все формы заявок (PhoneForm, LeadForm) подключена **Google reCAPTCHA v3** — пользователю ничего нажимать не нужно, проверка идёт в фоне.

## Что сделать

### 1. Ключи в Google reCAPTCHA

1. Зайдите на [Google reCAPTCHA Admin](https://www.google.com/recaptcha/admin).
2. Зарегистрируйте сайт: тип **reCAPTCHA v3**, домены вашего сайта.
3. Получите **Ключ сайта** (Site key) и **Секретный ключ** (Secret key).

### 2. Фронт (Vite)

В `.env` или `.env.local` в корне фронта добавьте:

```
VITE_RECAPTCHA_SITE_KEY=ваш_ключ_сайта
```

Пересоберите/перезапустите проект.

### 3. Бэкенд (облачная функция leads)

В настройках функции **leads** в Yandex Cloud добавьте переменную окружения:

```
RECAPTCHA_SECRET_KEY=ваш_секретный_ключ
```

Задеплойте функцию.

### 4. Поведение

- Если **не задать** ключи — формы работают как раньше, проверка капчи не выполняется.
- Если **задать оба** ключа — при отправке формы запрашивается токен reCAPTCHA v3 и отправляется на бэкенд. Бэкенд проверяет токен у Google; при score &lt; 0.3 заявка отклоняется с сообщением «Проверка не пройдена».
- Порог score (0.3) можно поменять в `backend/leads/index.py` — константа `RECAPTCHA_MIN_SCORE`.
