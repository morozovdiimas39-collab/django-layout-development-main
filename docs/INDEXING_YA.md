# Почему в поиске только главная (Яндекс)

## Что уже проверено и исправлено в коде

- **robots.txt** — разделы `/acting`, `/blog`, `/oratory` не закрыты от индексации.
- **noindex** — только у настроек (`/settings`); у статей блога и основных страниц стоит `index, follow`.
- **Canonical** — в `index.html` была опечатка в главном canonical (`https:/` вместо `https://`), исправлено.
- **Sitemap** — в проекте есть генерация sitemap (при сборке в `dist` попадает `sitemap.xml`); важно, чтобы по адресу сайта он реально отдаётся.

## Почему разделы могут не попадать в индекс

Сайт — **одностраничное приложение (SPA)** на React. При запросе любого URL (`/acting`, `/blog`, `/oratory`, статьи блога) сервер отдаёт один и тот же `index.html`. Заголовок, описание и контент подставляются уже в браузере через JavaScript (в т.ч. через Helmet).

Для Яндекса это значит:

1. **Контент виден только после выполнения JS.** Если бот не всегда или не полностью выполняет JS, он может не увидеть уникальные title/description и текст страницы и считать её пустой или дубликатом главной.
2. **Один и тот же начальный HTML для всех URL** может приводить к тому, что часть страниц не индексируется или индексируется с задержкой.

Поэтому «в поиске» часто оказывается только главная (она совпадает с дефолтным содержимым `index.html`), а разделы и блог — «загружено», но «в поиске» 0.

## Что сделать в Яндексе (Вебмастер)

1. **Проверить sitemap**  
   В [Яндекс.Вебмастер](https://webmaster.yandex.ru) → ваш сайт → «Индексирование» → «Файлы Sitemap» указать:  
   `https://казбек-меретуков.рф/sitemap.xml`  
   Убедиться, что файл доступен по этому адресу и в нём есть нужные URL (главная, /acting, /oratory, /blog, статьи).

2. **Переобход важных страниц**  
   «Индексирование» → «Переобход страниц» — добавить и отправить на переобход URL главных разделов, например:  
   - `https://казбек-меретуков.рф/acting`  
   - `https://казбек-меретуков.рф/oratory`  
   - `https://казбек-меретуков.рф/blog`  
   и несколько статей блога.

3. **Проверить, что видит робот**  
   В Вебмастере есть проверка «Как видит страницу робот» (или аналог). Проверить для `/acting` и одной статьи блога: видит ли робот нужный заголовок, описание и текст. Если видит только общий шаблон — причина в SPA и ограничениях отрисовки JS.

4. **Время**  
   После исправления canonical и при наличии рабочего sitemap индексация разделов может улучшиться в течение нескольких недель.

## Если нужно стабильно индексировать все страницы

Долгосрочный вариант — сделать так, чтобы по каждому URL сервер (или пререндер) отдавал уже готовый HTML с нужным title, description и основным текстом. То есть:

- **SSR (серверный рендер)** на React, или  
- **пререндер** ключевых страниц при сборке и отдача готового HTML по соответствующим путям.

Тогда робот будет видеть разный контент даже без выполнения JS, и шансы попадания разделов и блога в поиск вырастут.
