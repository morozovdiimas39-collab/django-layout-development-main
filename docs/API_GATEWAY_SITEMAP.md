# Sitemap и сайт на основном домене (казбек-меретуков.рф)

## Проблема

Если основной домен отдаёт только SPA (все пути → index.html), то `/sitemap.xml` даёт 404.

## Решение: домен через API Gateway

Нужно один раз направить трафик **казбек-меретуков.рф** в API Gateway. В шлюзе:
- запросы к **/sitemap.xml** обрабатывает функция sitemap (актуальный XML из БД);
- все остальные запросы отдаются из бакета (как сейчас).

### Шаги

1. **Открой файл** `api-gateway-main-domain.yaml` в корне репозитория.

2. **Замени плейсхолдеры:**
   - `BUCKET_NAME` — имя бакета, куда деплоится фронт (тот же, что в `YC_BUCKET` в GitHub Actions).
   - `SERVICE_ACCOUNT_ID` — ID сервисного аккаунта, от которого шлюз ходит в бакет (если при сохранении спеки будет 400 — укажи этот ID из консоли Yandex Cloud).

3. **Создай или отредактируй API Gateway** в консоли Yandex Cloud:  
   API Gateway → создать API / открыть существующий → вставить содержимое `api-gateway-main-domain.yaml` (уже с подставленными BUCKET_NAME и SERVICE_ACCOUNT_ID) → сохранить.

4. **Привяжи домен** казбек-меретуков.рф к этому API Gateway (в настройках домена в консоли укажи этот шлюз вместо бакета/CDN).

5. **Проверка:**  
   - https://казбек-меретуков.рф — открывается сайт.  
   - https://казбек-меретуков.рф/sitemap.xml — отдаётся XML.

После этого ничего дополнительно делать не нужно: деплой по-прежнему заливает фронт в бакет, sitemap всегда строится функцией по данным из БД.
