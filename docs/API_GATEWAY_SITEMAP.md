# Sitemap на своём домене (казбек-меретуков.рф/sitemap.xml)

Чтобы по адресу **https://казбек-меретуков.рф/sitemap.xml** всегда отдавался актуальный sitemap (из функции), нужен API Gateway в Yandex Cloud.

## Итоговая ссылка

**https://казбек-меретуков.рф/sitemap.xml** — один и тот же домен, ответ 200, XML из функции (всегда актуальный).

## Шаги в Yandex Cloud

1. **API Gateway**  
   Консоль Yandex Cloud → API Gateway → Создать API.

2. **Спецификация**  
   Вставить содержимое файла `api-gateway-sitemap.yaml` (корень репозитория) или только путь `GET /sitemap.xml` с интеграцией в функцию sitemap.  
   В интеграции указать `function_id`: **d4e970s0n7por7g0cpc3** (ID функции sitemap из func2url.json).

3. **Домен**  
   К созданному API привязать домен **казбек-меретуков.рф** (или тот, с которого должен открываться сайт).  
   Важно: если сейчас домен смотрит на бакет со статикой, нужно будет решить, кто обрабатывает запросы:
   - либо весь трафик идёт в API Gateway, а в нём настроены и статика (object_storage), и путь `/sitemap.xml` (cloud_functions);
   - либо домен остаётся на бакете, а для sitemap используется отдельный поддомен/другой способ (редко удобно).

4. **Проверка**  
   Открыть в браузере: `https://казбек-меретуков.рф/sitemap.xml` — должен открыться XML, не редирект и не 404.

## Формат ответа

Функция sitemap уже отдаёт `statusCode: 200`, `Content-Type: application/xml` и тело в формате sitemaps.org — API Gateway просто пробрасывает этот ответ по пути `/sitemap.xml`.
